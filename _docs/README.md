# home-0ps documentation

Documentation for the home-0ps GitOps Kubernetes lab. Four kinds of doc, plus the rolling review and the historical archive.

## Map

| Area | Where | What it's for |
| --- | --- | --- |
| **App guides** | `apps/` | How each live app is deployed, configured, backed up, and recovered |
| **Infra guides** | `infra/` | Per-domain reference for the platform layers |
| **Best practices** | [`guides/best-practices.md`](guides/best-practices.md) | The patterns + lessons learned — read before adding an app or changing infra |
| **Journey** | [`journey.md`](journey.md) | The narrative arc + retired-app retrospectives |
| **Decisions (ADRs)** | `decisions/` | The big architectural calls and their rationale |
| **Roadmap & Status** | [`roadmap.md`](roadmap.md) · [`status.md`](status.md) | Live open-items/sprints (from the reviews) and service status |
| **Reviews** | `reviews/` | The rolling state-of-the-lab tracker / open-items punch list (newest = current; regenerated by `/lab-review`) |
| **Archive** | `archive/` | Retired-app manifests + the original source docs the guides were distilled from |

## Conventions

- **Where's the current open-items list?** The newest `reviews/home-0ps-review-<date>.md`. Older ones are kept as the historical record.
- **ADR statuses:** Draft → Proposed → Accepted → Superseded. Superseded ADRs stay (with a pointer to what replaced them).
- **Archive is read-only history.** `archive/source-docs/` holds the long-form originals that the guides/ADRs were distilled from; `archive/<app>/` holds retired-app manifests.
